Compiling the Fwink Source Code

To compile Fwink, you will need the following tools:

  • MS Visual C++ 6.0
  • Visual Studio 6.0 Service Pack 5
  • MS Platform SDK February 2003 or later
  • DirectX 9.0 SDK (summer 2003 update) including C++ sample source code

Crossbar

You will need to copy crossbar.cpp and crossbar.h from the DirectX 9 SDK into the Fwink folder. If you installed the SDK to C:\DX90SDK, you will find these files in C:\DX90SDK\Samples\C++\DirectShow\Capture\amcap.

DirectShow Base Classes

You also need to build the Base Classes DirectShow sample. The project is located at C:\DX90SDK\Samples\C++\DirectShow\BaseClasses. Open the file baseclasses.dsw. Build the UNICODE release and debug versions of the project. It should create the libraries strmbase.lib and strmbasd.lib which Fwink needs to link to.

MS Layer For Unicode

The MS Layer for Unicode lets Unicode applications run on Windows 95, 98 and ME. Fwink will link with unicows.lib which replaces some of the Win32 Unicode functions. On Windows 95, 98 and ME, the unicows.dll library implements these functions. Windows NT, 2000 and XP will use their native system calls.

Warning: The February 2003 Platform SDK has a bug in unicows.lib that prevents the debug version of Fwink from linking in VC++ 6.0. This should be fixed with the next release of the Platform SDK. This issue should not affect Visual Studio .NET.
